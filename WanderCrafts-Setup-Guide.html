WanderCrafts PH — Pricing Calculator
Setup Guide for New Store Owners

This guide walks you through setting up your own copy of the WanderCrafts PH Pricing Calculator. You will need a GitHub account (for hosting) and a Supabase account (for data storage). Both are free.

Note: All your data (materials, products, orders, etc.) is stored in your own Supabase database — completely separate from other stores. Changes sync across all devices and browsers automatically.

Step 1 — Copy the App Files (GitHub)

  1. Go to github.com and create a free account if you don't have one.
  2. Create a new repository:
    • Click the + icon → New repository
    • Give it a name, e.g. my-store-calculator
    • Set it to Public
    • Click Create repository
  3. Upload these files to your new repository:
    • index.html
    • supabase-api.js
    • supabase-config.js
  4. Enable GitHub Pages:
    • Go to your repo → Settings → Pages
    • Under Source, select Deploy from a branch
    • Choose main branch → / (root) → click Save
  5. Your site URL will be:
    https://your-github-username.github.io/my-store-calculator/
Tip: It may take 1–2 minutes after saving for GitHub Pages to go live.

Step 2 — Create Your Supabase Database

  1. Go to supabase.com and sign up for a free account.
  2. Click New Project:
    • Enter a project name (e.g. my-store)
    • Set a database password (save this somewhere safe)
    • Choose the region closest to you
    • Click Create new project — wait about 1 minute for it to set up

Run the Setup SQL Script

  1. In your Supabase project, go to the left sidebar → SQL Editor
  2. Click New query
  3. Copy and paste the entire contents of the file supabase-setup.sql into the editor
  4. Click Run (or press Ctrl+Enter)
  5. You should see "Success. No rows returned" — this means all tables were created successfully

Create the Logo Storage Bucket

  1. In the left sidebar → Storage
  2. Click New bucket
  3. Name it exactly: Logos (capital L)
  4. Check Public bucket → click Save

Step 3 — Get Your Supabase Credentials

  1. In your Supabase project, go to Project Settings (gear icon, bottom left)
  2. Click API
  3. Copy two values:
    • Project URL — looks like https://abcdefgh.supabase.co
    • anon / public key — a long string starting with sb_publishable_...

Step 4 — Update the Config File

  1. Open the file supabase-config.js in a text editor (Notepad is fine)
  2. Replace the values with your own credentials:
const SUPABASE_CONFIG = {
  URL: 'https://your-project-id.supabase.co',
  ANON_KEY: 'your-anon-public-key'
};
  1. Save the file
  2. Upload the updated supabase-config.js to your GitHub repository (replace the old one)
Important: Never share your service_role key — only use the anon / public key in this file.

Step 5 — Open Your App

  1. Go to your GitHub Pages URL:
    https://your-github-username.github.io/my-store-calculator/
  2. Click the logo placeholder (top left) to upload your store logo
  3. Click the store name to edit it
  4. Click 🗄️ Sync Supabase to confirm everything is connected
  5. You should see ✅ Synced — you're ready to go!

Features Overview

Below is a summary of all available features in the app.

📦 Material

  • Add, edit, copy, and delete materials
  • Bulk upload via Excel/CSV template
  • Stock Status filter — defaults to In Stock on load; switch to Out of Stock or All using the dropdown
  • Date auto-conversion on bulk upload — any date format (e.g. 2-Jan, 1/2, Jan 2 2026) is automatically converted to YYYY-MM-DD

🔧 Equipment

  • Add, edit, and delete equipment
  • Bulk upload via Excel/CSV template with auto date conversion

⚡ Electricity Cost / 👷 Labor Cost

  • Add rates per kWh or per hour
  • Used as dropdown options in the Pricing Calculator

🧮 Pricing Calculator

  • Calculate cost price from Ink, Material, Electricity, Labor, and Packaging costs
  • Material Cost and Packaging Cost rows now show a Price/Qty field — dropdown shows product name only, price auto-fills in the field
  • If a material is Out of Stock, it still shows with saved price when editing (marked with ⚠️)
  • Set Markup, Discount, Tax to get Selling Price
  • Override with a Final Selling Price field
  • Save to Products or update existing products
  • After editing a product and clicking Update, automatically returns to the Product tab
  • All dropdowns sorted alphabetically

🖨 Ink Calculator

  • Supported printers: Brother T930DW, Canon IX6770, Epson L121, Other Printer
  • Page yields auto-fill when a printer is selected
  • Calculates Black/White and Colour cost per page
  • Save entries to use in the Pricing Calculator

🛍️ Product

  • View all saved products with Cost, Markup, Discount, Tax, Selling Price, Final Selling Price, Profit, Margin
  • View Recipe — shows breakdown of all costs with totals per category and Final Selling Price
  • Export to Excel or PDF (opens new window with Print button)
  • Edit, Copy, Delete products

📋 Order

  • Add, edit, copy, and delete orders
  • Bulk upload via Excel/CSV template with auto date conversion
  • Filter by Year, Payment Status, Order Status
  • Export to Excel or PDF

📈 Sales Overview

  • Monthly breakdown and top products summary
  • Filter by Year, Month, Payment, Order Status
  • Export to Excel or PDF

📊 Expense Overview

  • Combined view of Material and Equipment expenses
  • Filter by Year, Month, Type
  • Export to Excel or PDF

Troubleshooting

ProblemSolution
Button shows ❌ Retry SyncCheck that your URL and ANON_KEY in supabase-config.js are correct
Data not showing after refreshClick 🗄️ Sync Supabase to force a reload
Logo not savingMake sure the Storage bucket is named exactly Logos (capital L) and is set to Public
GitHub Pages not loadingWait 2–3 minutes after enabling Pages, then hard refresh (Ctrl+Shift+R)
Data not syncing across devicesAll devices must use the same GitHub Pages URL — data is tied to your Supabase project
